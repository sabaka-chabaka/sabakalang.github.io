Features

Everything you need. Nothing you don't.

⚡

Bytecode VM

Compiles to a compact instruction set and runs on a custom stack-based virtual machine with fast execution.

🔒

Static Typing

int, float, bool, string, arrays, structs, classes — all resolved at compile time with full type inference.

📦

Import System

Import .sabaka files and native .dll modules with alias namespacing. Deduplication built in.

🎨

SabakaUI

Build native WPF windows with a WebView2 bridge. Send data between JS and SabakaLang in real time.

🌍

Physics2D

Rigid body simulation, AABB and circle colliders, impulse resolution — all in pure SabakaLang.

📐

Canvas & Charts

2D drawing API and chart rendering. Histograms, line charts, scatter plots — render to any surface.

SabakaUI

Native UI.
Zero boilerplate.

SabakaUI wraps WPF and WebView2 into a two-line API. Write your interface in HTML/CSS/JS, then wire it to SabakaLang logic with ui.on() and ui.set().

JS → SabakaLang via sabaka.send(event, data)
SabakaLang → JS via ui.set(key, value)

SAR Packaging

Ship as a single file.

01 — Write
Source + manifest
Write your .sabaka files and a manifest.json declaring your entry point and DLL dependencies with aliases.
02 — Pack
SarPacker
SarPacker compiles to bytecode, resolves all DLL dependencies, and bundles assets, DLLs, and bytecode into a single .sar ZIP archive.
03 — Run
RuntimeEnvironment
SRE extracts the archive to a temp folder, loads DLLs with correct aliases, wires callbacks, and executes the bytecode on the VM.
